CAHIER D'AVANCEMENT - PROJET DATA VISUALIZATION
--------------------------------------------------------------------------------
Membres du groupe : Amine, Youssef, Joubrane, Amir
Sujet : Scouting Reports DataViz (Analyse de données footballistiques)
--------------------------------------------------------------------------------

[ 12/11 ] PHASE DE CONCEPTION ET ANALYSE CRITIQUE
-------------------------------------------------
Objectif : Définition du périmètre et analyse de l'existant.

> Livrable Tomuss :
Soumission individuelle d'une critique de visualisation de données (analyse des
types d'encodage, interactions et pertinence).

> Évolution du projet :
Nous avions initialement prévu de travailler sur des statistiques simples via
Excel.
DÉCISION MAJEURE : Après analyse, nous avons réalisé que cela limitait notre
potentiel de visualisation. Nous avons pivoté vers la recherche de données
événementielles précises (coordonnées X,Y sur le terrain, timestamps) pour
permettre des visualisations géographiques et tactiques avancées.


[ 09/12 ] ACQUISITION DES DONNÉES (BACKEND)
-------------------------------------------
Objectif : Mise en place du pipeline de données.

> Développement Python :
Création des scripts de web scraping (fichiers 'whoscored_scraper.py' et
'image_scraper.py') pour automatiser la récupération des données de matchs.

> Analyse de la structure :
Exploration des fichiers JSON récupérés (format des événements, IDs des joueurs).

> Nettoyage :
Traitement des données brutes pour ne garder que les informations pertinentes
(passes, tirs, actions défensives) en vue de leur exploitation par le futur
frontend.


[ 16/12 ] DÉVELOPPEMENT DU MVP (MINIMUM VIABLE PRODUCT)
-------------------------------------------------------
Objectif : Premier prototype fonctionnel.

> Architecture Web :
Mise en place de la structure HTML/CSS de base.

> Implémentation JS :
Développement des deux premières visualisations majeures pour valider notre
approche technique :
1. Heatmaps ('heatmap.js') : Pour visualiser la densité de présence d'un joueur
   sur le terrain.
2. Passmaps ('passmap.js') : Pour tracer les connexions et les trajectoires de
   balles.

> Résultat :
Nous avons une première version capable d'afficher les données d'un joueur
spécifique sur un terrain interactif.


[ 07/01 ] ENRICHISSEMENT ET DIVERSIFICATION
-------------------------------------------
Objectif : Finalisation des fonctionnalités de visualisation.

> Nouvelles Visualisations :
Pour offrir une analyse complète (Scouting Report), nous avons ajouté :
- Radar Charts ('radarChart.js') : Pour comparer les profils de joueurs.
- Shotmaps ('shotmap.js') : Localisation des tirs et buts.
- Analyse de Zones ('zoneAnalysis.js') : Découpage du terrain pour des
  statistiques sectorielles.
- Passes Progressives : Distinction des passes qui font avancer le jeu.

> Intégration :
Réunion de tous les modules dans le 'dashboard.js' pour une navigation fluide
entre les différents joueurs (Ounahi, Cherki, Doku, etc.).


[ 13/01 ] POLISSAGE ET DÉPLOIEMENT (BONUS)
------------------------------------------
Objectif : Rendu final et mise en ligne.

> UX/UI :
Améliorations finales du design ('style.css') pour rendre le tableau de bord
plus lisible et esthétique.

> Bonus technique :
Décision collective de ne pas se limiter au local. Configuration et déploiement
du site en ligne pour le rendre accessible publiquement et démontrer la
robustesse du projet.

> Tests finaux :
Vérification de la cohérence des données affichées par rapport aux fichiers
JSON sources.
